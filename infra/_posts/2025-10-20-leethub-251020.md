# 리트허브 연동하기
[리트코드](https://leetcode.com/)로 코딩 테스트 연습을 하기 위해서 리트코드로 문제를 풀던 도중 백준허브처럼 깃허브로 연동하여 관리할 수 있는지 찾아보게 되었다.

찾아보니 [LeetHub](chrome-extension://ghcjnhlmidpoegkdphbiknaindofkhad/src/html/welcome.html)에서 백준허브처럼 깃과 연동하여 사용할 수 있었다.

기존 백준허브로 사용하던 알고리즘 폴더와 연동하고 싶어, 기존 폴더이름으로 Link하였으나...
![leetHubImg](/assets/img/posts/infra/251020/img.png)
루트 안에 새 폴더가 적용되지 않고 LeetCode문제가 업로드되고 있었다.
![img.png](/assets/img/posts/infra/251020/img1.png)

`/LeetCode/001-문제` 이런식의 폴더 구조로 바꾸고 싶어서 튜닝을 결심.. 구글링하여 최근 블로그 게시글로 방법을 찾았다.

처음에는 v2.0가 가장 안정적?이라고 들어서 처음에는 v2를 받아서 사용하려 했지만, 개발자모드에서 업로드하려고 할 때마다 

> ⚠️ 매니페스트 파일이 없거나 읽을 수 없습니다 매니페스트를 로드할 수 없습니다.

라며 업로드시켜주지 않아 다른 버전을 사용했다.

> 해당 버전에는 `manifest-chrome.json`이 있었다. 그런데 chrome은 `manifest.json`만 인식한다길래 이름 바꿔보고 지피티랑 별짓 다했는데 그냥 버전 바꾸는게 훨씬 빨랐다.

사용한 버전은 v3.0이다. https://github.com/raphaelheinz/LeetHub-3.0

## Manual Install
`README.md`의 Install - Manual Install을 참고했다.
![manualInstall](/assets/img/posts/infra/251020/img2.png)

### 1. OAuth app 생성
시키는 대로 만들었다.
> Create your own OAuth app in GitHub (https://github.com/settings/applications/new) and store CLIENT_ID and CLIENT_SECRET confidentially
> - Application name: [CUSTOM]
> - Homepage URL: https://github.com/raphaelheinz/LeetHub-3.0
> - Authorization callback URL: https://github.com/

https://github.com/settings/applications/new 해당 링크에서 
![img_1.png](/assets/img/posts/infra/251020/img3.png)
이렇게 내용을 채우고 생성하면 `ClientID`가 생성되고, `Generate a new client secret`을 클릭하면 `Client secretes`이 생성된다.
![client](/assets/img/posts/infra/251020/img4.png)


### 2. repository 실행
> Download the project ZIP (Releases) or clone this repository
> Run npm run setup to install the developer dependencies

레포지토리를 클론받아 `npm run setup`을 실행했다.
이 과정은 시간이 조금 걸린 것 싶다.
![img.png](/assets/img/posts/infra/251020/img5.png)

### 3. OAuth app 정보 입력
> Update CLIENT_ID and CLIENT_SECRET in src/js/authorize.js and src/js/oauth2.js with your ids

파일을 열고 `src/js/authorize.js`, `src/js/oauth2.js`에서 각각 `CLIENT_ID`와 `CLIENT_SECRET`을 방금 발급받은 정보로 수정했다.

### 4. 업로드 위치 지정
이 부분은 다른 버전과 과정이 같다.

`leetCode.js`에서 `const URL`을 수정하면 된다.

내가 클론받은 버전에선 기존 URL이 주석으로 처리되어 있었다.
![img.png](/assets/img/posts/infra/251020/img6.png)

맨 위에 `constructGitHubPath`함수 에서 경로를 수정했다.

커스텀할 때 `/contents` 뒤에 넣고싶은 폴더 경로를 추가하는것 같았다. 그래서 `/contents` 뒤에 `/LeetCode/`경로를 추가했다.
![img.png](/assets/img/posts/infra/251020/img7.png)

### 5. 크롬 확장프로그램 업로드
크롬의 확장프로그램으로 들어가 개발자 모드를 켜고 `압축해제된 확장 프로그램을 로드`를 클릭하여 수정한 프로그램을 업로드한다.
![img.png](/assets/img/posts/infra/251020/img8.png)
이전과 다르게 매니페스트를 로드 오류 없이 업로드 된다.
![img.png](/assets/img/posts/infra/251020/img9.png)

### 6. 업로드 확인
리트허브 확장프로그램으로 들어가서 링크할 기존 알고리즘 폴더를 선택한다.

깃허브와 연동 후 리트코드로 돌아가 아무 문제나 업로드 시도했다.
![img.png](/assets/img/posts/infra/251020/img10.png)
기존에 알고리즘 폴더 루트에 문제가 업로드 되던 것과 달리, `/LeetCode`폴더가 생성되었고 그 안에 문제가 들어오는 것을 확인할 수 있다!
